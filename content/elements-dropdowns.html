# Dropdowns [https://en.wikipedia.org/wiki/Drop-down_list](https://en.wikipedia.org/wiki/Drop-down_list) ## Usage ## Position Dropdown menus should appear below the input for which they're meant to provide a value. In the case where there isn't enough space at the bottom of the page, dropdowns and other controls should appear above the input.

Primary Dropdowns

Dropdowns in Bionic are almost always implemented as comboboxes -- this controls allows the user to type in a value to search for it in a long list

The arrow should be the caret-down icon in font awesome: http://fontawesome.io/icon/caret-down/

    <span class="input-holder">
    <input class="ui-autocomplete-input-with-arrow ui-autocomplete-input hasAutoComplete" 
    type="text" autocomplete="off">
</span>

Secondary Dropdowns

In cases where multiple dropdowns are needed, using a secondary style helps provide additional contrast/separation between menus.

    <span class="input-holder">
    <input class="ui-autocomplete-input-with-arrow ui-autocomplete-input hasAutoComplete" 
    type="text" autocomplete="off">
</span>

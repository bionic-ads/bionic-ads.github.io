# Grids ## Layout ![Grid](images/content/components/grid.png) ## Grid Headers
  • Font Size: 13px
  • Font Weight: normal
  • Clickable/sortable: bionic blue color
  • Not Clickable/sortable: normal text (off black) color
## Grid Content
  • Font Size: 13px
  • Alternate Font Size: 11px (for secondary content)
  • Font Weight: normal
### Column Widths Bionic grids have two primary column width options, Fixed Width and Evenly Spaced. #### Fixed-Width (Default) Fixed-width grid layouts should use a fluid final column when there are only a handful of columns. EXAMPLE: Picklist Admin, Campaign List #### Evenly Spaced Evenly spaced columns are a useful option when columns make sense to be the same size. This is useful for data visualizations such as a Usage Report heatmap. EXAMPLE: Usage Report ------ ### Grid Line Colors Primary grid lines (headers and vertical dividing lines) should be 1px solid #dddddd; In Sass, the color is defined as a variable called #grid-primary-border-color Secondary grid lines (main body lines) should be 1px solid #eeeeee; In Sass, the color is defined as a variable called #grid-secondary-border-color ------ ### Fixed First Column The first column (after the line numbers) should have a border on the right so that the border is visible on scroll ------ ### Summary Rows Should appear outside the grid and use bold text formatting in most cases ------ ### Row Height The height of the grid rows (headers and main content) should be 40px ## Filtering Any filtering of a grid should persist at the user level. See [Filters](#/components-filters) for more information. ## Actions There are two types of Bionioc components where actions are performed. 1. Main Menu 1. Action Menu ## Sorting To add sorting to a grid column header, add a data sort ID and direction to the header cell's span element. 
        <div class="nxm-td grid-row-bionic-date">
            <span class="" data-sort-id="BILLDATE" data-sort-direction="descending">
                Bill Date
            </span>
        </div>
### Include Sorting Icons Where Applicable Allow sorting for icon columns: e.g., Following, Locked, Active ### Names Sort the names of people using First Name instead of Last Name (presuming the first name is what's shown first) ## Grid Details ![Grid Details](images/content/grid-details.png) ### Grid Row Class

![Grid Number](images/content/grid-numbering.png) ### Row Numbering Classes This code will populate a row cell with the row's sequential index. 
        <div class="nxm-td grid-row-number">

        </div>
![Grid Checkbox](images/content/grid-checkbox.png) ### Row Checkbox Classes This code will generate a checkbox for the entire row. Replace 'model' with the variable name of the model passed to the component. 
        <div class="nxm-td grid-row-select ">
            <div class="bionic-checkbox">
                <input type="checkbox" 
                name="name-<%= model.nextmarkId %>" 
                id="<%= model.nextmarkId %>" 
                class="nxm-grid-select-row" 
                <%= insured(model, 'selected', false)? 'checked' : '' %> >
                <label for="name-<%= model.nextmarkId %>">
                    <span class="box checkbox"></span>
                </label>
            </div>
        </div>
